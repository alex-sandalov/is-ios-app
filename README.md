# Банк **beta-bank**

## Тема

Приложение включает 3 основных экрана:
1. Экран авторизации
2. Экран списка счетов, карт и вкладов
3. Экран детальной информации о выбранном продукте

---

## Выбранная архитектура
Выбрана архитектура **MVP**.

### Обоснование выбора
1. **По сравнению с MVC**, в MVP логика обработки пользовательских действий и подготовки состояния экрана выносится из `UIViewController` в `Presenter`. Это уменьшает связность между UI и бизнес-логикой.

2. **По сравнению с MVVM**, MVP не требует механизма биндинга между состоянием и представлением. Для UIKit-приложения это позволяет описать взаимодействие через обычные протоколы `View` и `Presenter`.

3. **По сравнению с VIPER**, MVP содержит меньше обязательных сущностей и связей между ними. Для приложения из трёх экранов это позволяет сохранить модульность без избыточного усложнения структуры проекта.

4. **По сравнению с Clean Swift / YARCH**, MVP требует меньше вспомогательных типов и шаблонных сущностей. Для учебной работы это делает структуру приложения более компактной и понятной.

5. **С точки зрения тестирования**, MVP позволяет тестировать `Presenter` отдельно от `View`, так как взаимодействие между слоями задаётся через протоколы. Это упрощает подмену зависимостей мок-объектами.

## Список модулей и их ответственности

### 1. Auth
Отвечает за авторизацию пользователя по логину и паролю, валидацию введённых данных, запуск сценария входа и переход к экрану со списком продуктов после успешной авторизации.

### 2. ProductsList
Отвечает за отображение списка продуктов пользователя: счетов, карт и вкладов. Управляет загрузкой данных, отображением состояний экрана, обработкой ошибок, обновлением списка и выбором конкретного продукта.

### 3. ProductDetails
Отвечает за отображение подробной информации о выбранном продукте. Показывает основные данные по счёту, карте или вкладу: баланс, статус, реквизиты, дату открытия, а также список последних операций и доступных действий.

## Экран 1 — Auth

### Вход модуля

```swift
struct AuthModuleInput: Equatable {
    let prefilledLogin: String?
}
```

### Выход модуля

```swift
enum AuthModuleOutput: Equatable {
    case didAuthorize(UserSession)
}
```

### Состояния UI

```swift
enum AuthViewState: Equatable {
    case idle
    case loading
    case error(message: String)
    case authorized
}
```

### Основные сценарии

1. **Успешная авторизация**
   - пользователь открывает экран
   - вводит логин и пароль
   - нажимает кнопку входа
   - выполняется авторизация
   - открывается экран списка продуктов

2. **Ошибка валидации**
   - пользователь открывает экран
   - вводит некорректные данные или оставляет поля пустыми
   - нажимает кнопку входа
   - экран отображает сообщение об ошибке

3. **Ошибка авторизации**
   - пользователь вводит корректные данные
   - нажимает кнопку входа
   - выполняется запрос на авторизацию
   - при ошибке экран отображает сообщение пользователю

## Экран 2 — ProductsList

### Вход модуля

```swift
struct ProductsListModuleInput: Equatable {
    let session: UserSession
}
```

### Выход модуля

```swift
enum ProductsListModuleOutput: Equatable {
    case didSelectProduct(id: String)
    case didRequestLogout
}
```

### Состояния UI

```swift
enum ProductsListViewState: Equatable {
    case idle
    case loading
    case content([ProductListItem])
    case empty(message: String)
    case error(message: String)
}
```

### Основные сценарии

1. **Успешная загрузка списка**
   - пользователь переходит на экран после авторизации
   - выполняется загрузка списка продуктов
   - экран отображает счета, карты и вклады пользователя

2. **Пустой список**
   - пользователь открывает экран
   - выполняется загрузка данных
   - если продукты отсутствуют, экран отображает пустое состояние

3. **Ошибка загрузки**
   - пользователь открывает экран
   - выполняется запрос на получение списка продуктов
   - при ошибке экран отображает сообщение пользователю

4. **Переход на детальный экран**
   - пользователь выбирает элемент списка
   - передаётся идентификатор выбранного продукта
   - открывается экран детальной информации

---

## Экран 3 — ProductDetails

### Вход модуля

```swift
struct ProductDetailsModuleInput: Equatable {
    let productId: String
    let session: UserSession
}
```

### Выход модуля

```swift
enum ProductDetailsModuleOutput: Equatable {
    case didClose
}
```

### Состояния UI

```swift
enum ProductDetailsViewState: Equatable {
    case idle
    case loading
    case content(ProductDetails)
    case error(message: String)
}
```

### Основные сценарии

1. **Успешная загрузка информации**
   - пользователь открывает выбранный продукт
   - выполняется загрузка данных по продукту
   - экран отображает подробную информацию

2. **Ошибка загрузки**
   - пользователь переходит на экран деталей
   - выполняется запрос данных
   - при ошибке экран отображает сообщение пользователю

3. **Возврат к списку**
   - пользователь закрывает экран деталей
   - происходит возврат к экрану списка продуктов
